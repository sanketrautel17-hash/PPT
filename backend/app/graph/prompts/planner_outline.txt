## Instructions
1. Decide the total number of slides (between {min_slides} and {max_slides}). Aim for a COMPLETE, thorough presentation — use as many of the available unique layouts as needed to fully address the topic.
2. Select one layout per slide from the AVAILABLE LAYOUTS list below.
3. Assign a clear purpose to each slide. Every purpose MUST reference the actual company/topic in the user request — do NOT use generic or template placeholder descriptions.
4. Follow the presentation narrative arc: Opening → Context → Data/Insights → Action/Conclusion → Closing.
5. Do NOT use guidance slides (classification = "guidance") as layouts.
6. Each layout variant (template_slide_index) should be used at most ONCE. Select the most relevant unique layouts to cover all key sections of the presentation.
7. Any closing/thank-you/Q&A slide MUST be the final item in the array. Never place it in the middle.
8. The JSON array order IS the final slide order. Put slides exactly in the sequence they should appear in the PPT.

## Output Format
